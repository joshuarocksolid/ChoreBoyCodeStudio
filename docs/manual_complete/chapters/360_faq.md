# Frequently Asked Questions

Short answers to common questions. Each links to the chapter with full detail.

## General

**What is ChoreBoy Code Studio?**
A complete, project-first environment for writing, running, and debugging Python on a
ChoreBoy appliance. See "What is ChoreBoy Code Studio?".

**Do I need the internet to use it?**
No. It runs entirely on the appliance with no internet, no terminal, and no extra
installation.

**Where are my projects stored?**
Wherever you create them — a project is an ordinary folder. The application adds a visible
`cbcs/` sub-folder for metadata. See "File & folder reference".

## Projects and files

**How do I start a new project?**
**File > New Project from Template...**. See "Projects: open, create, import".

**Can I open a folder that wasn't made by Code Studio?**
Yes. Open any folder of Python files; the application creates `cbcs/project.json`
automatically. See "Projects: open, create, import".

**How do I change which file runs with Run Project?**
Right-click the file in the Explorer and choose **Set as Entry Point**.

**I deleted a file by mistake — can I get it back?**
Yes. Deleted items go to a recoverable trash, and **File > Open Global History...** can
restore files from Local History. See "Local History & recovery".

## Running and debugging

**Why does my program run in a separate window/process?**
For safety. Your code runs in a separate runner process so a crash never takes down the
editor. See "How it works".

**How do I pass command-line arguments to my program?**
Use **Run > Run With Arguments...** for a one-off, or **Run Configurations...** to save
them. See "Running code".

**My breakpoints don't pause — why?**
This can be an environment limitation. Use a normal Run and read the traceback as a
fallback. See "Debugging" and "Troubleshooting by symptom".

**Can I run FreeCAD GUI macros here?**
Runs are headless. Edit/save here, then run GUI/document macros inside FreeCAD. See
"FreeCAD workflows & headless limits".

## Editing and intelligence

**Does completion run my code?**
No. Editor intelligence is static and never executes your project code. The Python Console
is different — it inspects live objects on purpose. See "Code intelligence".

**How do I format my code?**
**Tools > Format Current File** (Black), and **Organize Imports**. Configure via
`pyproject.toml`. See "Python formatting & imports".

**How do I rename something everywhere?**
Place the cursor on it and press `F2` (Rename Symbol). You get a preview before applying.
See "Code intelligence".

## Settings and customization

**How do I change the theme?**
**View > Theme** — five modes including two High Contrast modes. See "Themes in depth".

**Can settings differ per project?**
Yes, for many sections. Global settings apply by default; project settings override them.
See "Settings overview".

**How do I change a keyboard shortcut?**
**File > Settings... > Keybindings**. See "Keyboard shortcuts".

## Dependencies, plugins, packaging

**How do I add a Python package?**
**Tools > Add Dependency...** from a local wheel, zip, or folder. See "Managing
dependencies".

**How do I share a finished project?**
Use **Package Project** for an installable build, or just copy/zip the folder for a
backup. See "Packaging, sharing & installing".

**A plugin is causing problems — how do I recover?**
Disable it, or use **Safe mode**, in the Plugin Manager. See "Using plugins".

## Support

**How do I get help with a problem?**
Generate a **Support Bundle** (**Tools > Generate Support Bundle**) and share it. See
"Diagnostics & support tools".

**Where are the logs?**
**Help > Open Application Log**, or `~/choreboy_code_studio_state/logs/app.log`. Per-run
logs are in each project's `cbcs/logs/`.
